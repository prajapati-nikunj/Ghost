# Story Breakdown

## Purpose
Decompose features or requirements into actionable user stories.

## Workflow
1. Analyze the feature or requirements document.
2. Identify independent units of work.
3. Define "As a [role], I want [action], so that [value]" for each story.
4. Set clear Acceptance Criteria (AC).

## Expected Output
A list of user stories with AC, ready for ticket creation.
