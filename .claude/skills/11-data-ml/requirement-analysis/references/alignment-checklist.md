# Alignment Checklist

## Pre-Execution Gate
Before writing any code, confirm each item with the user:

### 1. Scope
- [ ] Features list is finalized and prioritized (MoSCoW).
- [ ] Out-of-scope items are explicitly stated.
- [ ] MVP vs Full Product boundaries are clear.

### 2. Technical Stack
- [ ] Frontend framework confirmed.
- [ ] Backend framework confirmed.
- [ ] Database technology confirmed.
- [ ] Hosting/cloud provider confirmed.
- [ ] Mobile platforms (if any) confirmed.

### 3. Architecture
- [ ] Monolith vs Microservices decision made.
- [ ] API style decided (REST, GraphQL, gRPC).
- [ ] Authentication strategy agreed (JWT, OAuth2, SSO).
- [ ] Deployment strategy agreed (containers, serverless, PaaS).

### 4. Quality
- [ ] Test coverage expectations set.
- [ ] Performance SLAs defined (response time, uptime).
- [ ] Security requirements identified.

### 5. Process
- [ ] Communication cadence agreed.
- [ ] Review/approval gates identified.
- [ ] Definition of Done confirmed.

## Post-Alignment Statement
> "Based on our discussion, I will build [WHAT] using [TECH STACK], following [ARCHITECTURE], delivered in [TIMELINE], with [QUALITY GATES]. Do you approve this plan?"
